### 硬件准备
每个电机要裸露出顶部。重新校准时，都给那个电机接上单独的can口和供电（电池+闸开关），can口芯片使用对应肢体的can口芯片，防止混乱。

### 指令输入须知
所有指令输入，都要确保当前文件夹是`berkeley_humanoid_lite_lowlevel`。
（输入部分文件名之后按tab自动补全）

### 打开can口通信
每次重新插上can口usb线之后，都要重新打开can口通信
```sh
source scripts/start_can_transports.sh
```
因为只连了一个can口芯片，所以会报错三个无法连接，没报错的那个就是已经连接的。记住已经连接的can口芯片的名称。

### 检查电机id
接下来要检查已经连接的电机是哪个id。
**需要给电机上电。** 没上电的时候运行ping脚本会报错Error Frame。
```sh
uv run scripts/motor/ping_all.py can口名称
```

### 执行电气校准
现在知道连接的can口名称和电机id了，开始运行电气校准。
```sh
uv run scripts/motor/calibrate_electrical_offset.py -c can口名称 -i 电机id
```
例如，对于can_left_arm和id为7的电机，应该使用：`uv run scripts/motor/calibrate_electrical_offset.py -c can_left_arm -i 7`

运行之后，等待一会儿，电机应该**先逆时针**旋转一圈，**再顺时针**旋转一圈。
如果旋转顺序不一样，则应该**交换电机三相线中的任意两个**。断电操作，重新焊接在电机控制版上的三根黑线。
更换完毕之后再重新上电，执行电气校准。看到先逆时针再顺时针旋转就代表成功。

### 运行自动测试
校准成功后，运行左右摇摆测试脚本：
```sh
uv run scripts/motor/move_actuator.py -c can口名称 -i 电机id
```
运行成功后，终端不断打印当前电机角度和速度，电机会自动左右摇摆旋转。
按Ctrl+C停止。

这就完成了一个电机的重新校准。

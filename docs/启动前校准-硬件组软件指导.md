# 启动前校准 硬件组软件指导
在berkeley_humanoid_lite_lowlevel文件夹下打开终端。（打文件路径时，输入开头几个字母之后按tab可以自动补全）

每次can口的usb拓展坞连接之后，都要执行`source scripts/start_can_transports.sh`把can口打开。

## 普通测试步骤
使用`uv run scripts/motor/ping_all.py` ping通所有can口的电机

使用`uv run scripts/motor/calibrate_electrical_offset_multi.py -g can_left_leg:1,3,5,7,11,13 -g can_right_leg:2,4,6,8,12,14 -g can_left_arm:1,3,5,7,9 -g can_right_arm:2,4,6,8,10` 同时让四肢转动

## 连续测试（校准）步骤
硬件连线不稳导致的报错，需要在连续读取can口数据的时候才能检测出来。

插上手柄、IMU小板子之后，运行`uv run scripts/calibrate_joints.py`，会不断输出数字。
第一个数字是时间，后面的12个数字是每条腿的6个电机的校准值。

脚本运行之后，要手动拧每个关节，转动到两个方向的极限位置。转动的时候，可以看到屏幕上有数字会变化。
像坐位体前屈推小滑片一样，摆过那个角度校准过那个关节之后，同一个数字就不会再变了。

如果连线不稳，偶尔会显示`<can_left_leg> Error Frame: 12, 8`这种文字，说明对应部位连线不稳。
如果报错太多，脚本可能会卡死，按几下ctrl+c把脚本关掉。
如果报错太多导致can口离线了，那么脚本会刷屏`Error Frame: 76, 8`，这时候按几下ctrl+c关掉脚本，把can口usb扩展坞拔了重插，再重新执行`source scripts/start_can_transports.sh`把can口重新打开。
